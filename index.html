5 Great Bash Commands
ln clip scp tail find
ln

ln creates hard or symbolic links to files and directories.

A common use case for ln is, on a web server, to create a symbolic link of the site's source in the server's webroot.

The syntax for this operation is:


ln -s /full/path/to/source /full/path/to/destination
clip

The clip command can copy the output of a command to the clipboard or write the contents of the clipboard to a file.

The syntax to copy the output of a command is:


[command] | clip

The syntax to copy the clipboard to a file is:


clip > [filename]
scp

The scp command, or secure copy, allows files to be copied over ssh.

It can copy files in the following ways:

  • From the local host to a remote host
  • From a remote host to the local host
  • From a remote host to another remote host

An example of copying a file from the local host to a remote host is:


scp file.txt username@remotehost.com:/some/remote/directory
tail

The tail command reads the last few lines of a file to standard output.

One use case for the tail command is to watch all the files in a directory for changes:


tail -f *

This may be useful for many things, including:

  • Log files
  • Other output
find

While the ls command generally gets all the love, the find command is also useful if you're not sure where a file is or want to get a feel for the directory structure.

An example of the find command executed on the current directory that contains some directories with files in them will yield:


.

./dir1/panel-console.png
./dir1/panel-elements.png
./dir1/panel-network.png
./dir1/panel-remote-console.png
./dir1/panel-remote.png
./dir1/panel-resources.png
./dir1/panel-timeline.png
./dir2
./dir2/file3
./dir3
./dir3/file4
./dir3/file5
./dir3/file6

The output of find is also cleaner than that of ls for use as input to another command

The following will execute the identify command on all files in the previous example's dir1 directory:


find dir1 -maxdepth 1 -type f | xargs identify
